# 🧮 Calculator App

Простой консольный калькулятор на Python с историей вычислений.

## Возможности

- ➕ **Сложение**
- ➖ **Вычитание**
- ✖️ **Умножение**
- ➗ **Деление** (с защитой от деления на ноль)
- 🔼 **Возведение в степень**
- √ **Квадратный корень** (с защитой от отрицательных чисел)
- 📜 **История вычислений** — последние 10 операций (клавиша `h`)
- 🛡️ **Обработка ошибок** — неправильный ввод, деление на ноль, отрицательный корень

## Как запустить

```bash
python3 calculator.py
```

# 🌿 Git: как мы работали с ветками

Ниже — объяснение всего процесса, который мы прошли при добавлении
истории вычислений. Все термины — простыми словами.

---

## Что такое Git и зачем он нужен

Git — это система контроля версий. Представь, что ты пишешь код
и хочешь:

- Сохранять все версии проекта (как сохранения в игре)
- Пробовать новые фичи, не ломая рабочую версию
- Работать вдвоём/втроём над одним проектом
- Откатываться назад, если что-то сломалось

Git запоминает **каждое изменение** в специальной базе внутри папки `.git`.

---

## Основные понятия

### Репозиторий (repository, repo)

Папка с проектом, за которой следит Git. У тебя это `calculator-app/`.
Внутри есть скрытая папка `.git` — там Git хранит всю историю.

> **Где лежит:** https://github.com/9KDA0/calculator-app

---

### Коммит (commit)

**Снимок проекта** в определённый момент времени. Когда ты сделал
изменения и они тебя устраивают — делаешь коммит.

```
git add -A            # добавляем все изменённые файлы в «корзину»
git commit -m "..."   # сохраняем снимок с описанием
```

Каждый коммит имеет:
- Уникальный ID (например, `75a006a`)
- Автора (ты — `9KDA0`)
- Сообщение (что сделано)
- Ссылку на предыдущий коммит → получается цепочка

```
75a006a  ●  Initial commit: простой калькулятор на Python
    │
4e6bd3f  ●  feat: добавлена история вычислений
    │
b8d919f  ●  Merge pull request #1 (слияние веток)
```

---

### Ветка (branch)

**Независимая линия разработки.** Когда ты создаёшь ветку —
ты делаешь «копию» проекта и можешь менять в ней что угодно,
не затрагивая основную версию.

```
master ───●───●───●   ← основная ветка (всегда рабочая)
                \
feature/history  ●───●   ← эксперименты, новые фичи
```

**Зачем:** ты можешь полдня делать новую фичу, а если она сломается —
просто удаляешь ветку и `master` остаётся нетронутым.

Основные команды:
```bash
git branch feature/history   # создать ветку
git checkout feature/history # переключиться на неё
git checkout master          # вернуться обратно
```

---

### Pull Request (PR, пул-реквест)

**Формальный запрос на вливание.** Когда фича в ветке готова,
ты создаёшь PR:

> «Я сделал историю вычислений в ветке `feature/history`.
> Проверьте и влейте это в `master`.»

PR создаётся на GitHub (через сайт или `gh pr create`).
Там видно:
- Какие файлы изменились (diff)
- Обсуждение (комментарии)
- Статус (open → merged → closed)

В нашем проекте PR №1: **feat: добавлена история вычислений** → слит.

---

### Merge (слияние, вливание)

**Объединение веток.** Когда PR одобрен — делается merge.
Git берёт все изменения из `feature/history` и применяет их к `master`.

```
ДО merge:                      ПОСЛЕ merge:

master ───●                    master ───●───●───●  ← история теперь в master
                \                            /
feature/history  ●───●         feature/history  ●───●  (потом удалена)
```

Результат: код из ветки теперь в основной версии. Ветку можно удалить —
она больше не нужна.

---

### Push (отправка) и Pull (получение)

- **Push** — отправить свои коммиты с компьютера на GitHub (в облако)
- **Pull** — скачать чужие коммиты с GitHub на компьютер

```bash
git push origin master           # отправил → теперь все видят
git pull origin master           # скачал → теперь у тебя актуальная версия
```

`origin` — это стандартное имя для ссылки на GitHub-репозиторий.

## Полный цикл: как мы добавили историю

| # | Шаг | Команда | Что произошло |
|---|-----|---------|---------------|
| 1 | Создали ветку | `git branch feature/history` | Появилась песочница для экспериментов |
| 2 | Переключились | `git checkout feature/history` | Теперь мы внутри ветки |
| 3 | Написали код | Правки в `calculator.py` | Добавили историю, клавишу `h`, `deque` |
| 4 | Закоммитили | `git commit -m "feat: ..."` | Сохранили снимок в ветке |
| 5 | Отправили | `git push -u origin feature/history` | Ветка улетела на GitHub |
| 6 | Создали PR | `gh pr create --base master` | Запросили вливание в master |
| 7 | Влили | `gh pr merge --merge` | Изменения попали в master |
| 8 | Удалили ветку | `--delete-branch` | Ветка больше не нужна |

## Шпаргалка по командам

```bash
# Посмотреть состояние
git status                  # что изменилось?
git log --oneline           # история коммитов
git branch -a               # все ветки (локальные + удалённые)

# Работа с ветками
git branch имя-ветки        # создать ветку
git checkout имя-ветки      # переключиться
git branch -d имя-ветки     # удалить локально

# Сохранение и отправка
git add -A                  # добавить все изменения
git commit -m "сообщение"   # сделать коммит
git push origin master      # отправить на GitHub
git pull origin master      # скачать с GitHub

# Pull Request (через GitHub CLI)
gh pr create --base master --head моя-ветка   # создать PR
gh pr merge --merge                            # влить PR
gh pr list                                     # посмотреть список PR
```

## Ошибки и защита

Калькулятор обрабатывает все основные ошибки:

| Ситуация | Результат |
|----------|-----------|
| Деление на ноль | `❌ Деление на ноль невозможно!` |
| Корень из отрицательного | `❌ Нельзя извлечь квадратный корень...` |
| Ввод текста вместо числа | Повторный запрос: «введите корректное число» |
| Неверный номер операции | `❌ Неверный выбор. Введите число от 0 до 6, или h.` |
| Ctrl+C в любой момент | `👋 До свидания!` — выход без ошибки |
